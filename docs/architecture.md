# 格物 Gewu 架构与设计决策

> 配合 [docs/research/](research/) 阅读：每个决策都对应调研中的已验证发现（F1-F11）。

## 1. 总体数据流

```
symbol + as_of
   │
   ▼
DataService.load_bundle()          # 数据层
   ├─ DataCache（parquet + 元数据，过期重抓、失败退旧）
   ├─ 多源降级抓取（akshare_source / yahoo_source）
   └─ PIT 过滤（pit.py 法定披露日规则）→ 预计算摘要（指标/分位/增速）
   │
   ▼
render_all()                       # 上下文渲染：LLM 唯一的世界接口
   │
   ▼
LangGraph StateGraph               # agents/graph.py
   START → [基本面 ∥ 技术面 ∥ 舆情 ∥ 估值] →(屏障)→ 多头 ⇄ 空头(N轮) → 研究主管 → END
   │
   ▼
build_report() → fact_check.audit()   # 研报装配 + 数字溯源审计（确定性）
```

## 2. 数据层

### 2.1 多源降级（实测驱动）

2026-06 实测发现东财 push2 系行情接口在部分网络环境直接拒绝连接（`RemoteDisconnected`），
而同属东财的数据中心接口（datacenter-web，估值）与搜索接口（新闻）正常。因此：

| 数据集 | 主源 | 降级 | 实测状态 |
|---|---|---|---|
| 日线行情 | 东财 `stock_zh_a_hist` | 新浪 `stock_zh_a_daily` → 腾讯 `stock_zh_a_hist_tx` | 东财被拒时新浪/腾讯可用 |
| 财务指标 | 新浪 `stock_financial_analysis_indicator` | — | ✅ |
| 财务摘要 | 同花顺口径 `stock_financial_abstract` | — | ✅ |
| 估值序列 | 东财数据中心 `stock_value_em` | 百度 `stock_zh_valuation_baidu` | ✅ |
| 个股新闻 | 东财 `stock_news_em` | — | ✅（近期约百条） |
| 公司概况 | 巨潮 `stock_profile_cninfo` | 东财 `stock_individual_info_em` | 巨潮可用 |
| 公告列表/公告日/PDF | 巨潮官方 API（`cninfo_source.py` 直连） | 公告日缺失退法定截止日 | ✅（注意时间戳为 UTC 毫秒，须转北京时间） |
| 美股财务/EPS | SEC EDGAR companyfacts（真实 `filed` 日） | yfinance + 90 天保守滞后 | data.sec.gov 对部分地区 IP 403，自动降级 |

注意事项：

- 各源成交量单位不一致（手/股），单源内部一致 → 只用相对量（量比），不跨源比绝对值；
- `request_interval` 模块级节流：任意两次上游请求（含成功路径、跨数据集）至少间隔该秒数；
- 缓存原子写入（临时文件 + `os.replace`），损坏缓存自动删除重抓；抓取失败时退回过期缓存并标记 `stale`，研报中如实披露；
- 已知限制：缓存无跨进程文件锁（并发进程各自完整写入，靠 `os.replace` 保证读到完整版本）；前复权价格以抓取日为锚，跨抓取日的绝对价位不可比（评测只用收益率，不受影响）。

### 2.2 pandas 3 + pyarrow 兼容性坑

pandas 3 安装 pyarrow 后默认用 Arrow(RE2) 字符串引擎，akshare 内部正则含 RE2
不支持的 `\u` 转义（实测 `stock_news_em` 抛 `ArrowInvalid`）。在 `akshare_source.py`
模块入口强制 `pd.set_option("mode.string_storage", "python")` 回退。

### 2.3 Point-in-Time：真实公告日优先，法定披露日兜底

财报可见日期的判定优先级：
1. **真实公告日**（A股：巨潮定期报告公告时间；美股：EDGAR `filed` 申报日）；
2. 缺失时退回**法定披露截止日**——任何数据缺口只会让评测更保守，绝不泄漏未来。

两个已被回归测试钉住的坑：巨潮 `announcementTime` 是 UTC 毫秒 epoch，不转北京时间
会把公告日早算一天（8 小时前视窗口）；EDGAR 同一报告期存在修正件（10-K/A），
必须取最早 `filed` 的原始申报。

A股法定披露截止日：

| 报告期末 | 可见日期 |
|---|---|
| 03-31（一季报） | 当年 04-30 |
| 06-30（半年报） | 当年 08-31 |
| 09-30（三季报） | 当年 10-31 |
| 12-31（年报） | **次年** 04-30 |

性质：可能晚于真实公告日（agent 比真实世界晚看到财报）→ 评测**偏保守而非偏乐观**，
绝不泄漏未来。这是在免费数据约束下能做到的最诚实近似；真实公告日（巨潮）在路线图上。

行情/估值按自然日截断；技术指标在**截断后**的序列上计算（测试 `test_technical_indicators_follow_truncation` 守护）。

**公司概况的特殊处理**：profile（巨潮/东财/yfinance）是抓取当日的快照，无法回溯。
历史模式（as_of < 今天）下做最小化处理：剔除入选指数与主营业务/经营范围（前者直接泄漏
"未来入选沪深300"，后者反映 as_of 之后的转型），并剥离股票简称中的 `*ST/N/退` 等风险
前缀（否则"未来才戴帽"的事实会进入回测上下文）；行业等慢变字段保留但在 warnings 与
回测局限声明中如实标注残余前视风险。

## 3. Agent 层

### 3.1 为什么是「分析师团队 + 辩论 + 主管」

TradingAgents（85k stars, arXiv:2412.20138）验证了该编排范式的社区认可度；
格物面向投研而非交易，故将末端的"交易员/组合经理"替换为"研究主管 + 溯源审计"，
产出物从交易信号变为**可审计的研报**。

### 3.2 幻觉数字的三道防线

1. **预计算**：所有数字（指标、分位、增速）由代码算好注入上下文，LLM 不做算术；
2. **提示词纪律**：所有角色的 system prompt 都带 DATA_DISCIPLINE（只许引用上下文数字、保持原值与单位）；
3. **事后审计**（`fact_check.py`，确定性、无 LLM）：
   - 对照语料 = `render_all()` 渲染给 LLM 的同一份文本（所见即所审）；
   - 实质性数字判定：有小数点 / 带单位（%元倍亿万手股点）/ ≥100；豁免年份、日期、指数名（沪深300）、叙述性小整数；
   - 匹配：绝对值匹配（负号归一）+ 舍入感知（按报告小数位）+ 0.5% 相对容差 + 万/亿/万亿单位展开；
   - 结果（溯源率 + 不可溯源清单）作为附录写进每份研报。

审计范围由 `<!--audit:start/end-->` 标记界定：只审 LLM 生成的正文，
代码生成的元数据（评级徽章、数据源表）不计入。

### 3.3 状态设计

DataFrame 不进图状态。节点是持有（llm, 渲染后上下文）的闭包，状态里只有文本与结论
（`analyst_reports` 用 dict-merge reducer 支持四分析师并行写入，`debate` 用 list-append）。
好处：状态可序列化、可检查、辩论轮数可配置且有测试守护。

## 4. 评测层（evaluate/）

设计完全基于 FINSABER（KDD 2026, arXiv:2505.07078）对该领域评测缺陷的批判：

| FINSABER 指出的缺陷 | 格物的对策 |
|---|---|
| 数据泄漏（agent 看到未来） | 评测走与生产**同一条** `ResearchPipeline` 路径，PIT 由数据层强制 |
| 幸存者偏差（只测活下来的股票） | `--universe csi300` 用 baostock 取 **start 日期的历史成分**（含此后退市股） |
| 数据窥探（挑表现好的标的/窗口） | 固定 seed 的可复现抽样；评测日按固定频率生成 |
| 无基线对照 | 动量（60日）与买入持有基线永远同 panel 输出 |
| 夸大结论 | 汇总报告强制附带局限性声明（`LIMITATIONS`） |

指标：非中性评级方向命中率、期收益面板的 CR/年化/夏普/最大回撤、评级分布、平均溯源率。

## 5. LLM 层

单一 `LLM` 协议（`complete(system, user, json_mode)`）：

- `ChatLLM`：OpenAI 兼容客户端，默认 DeepSeek（调研结论：模型接入已商品化，不作为卖点），指数退避重试，JSON 模式 + 容错解析（`extract_json` 剥围栏/提取首个对象）；
- `MockLLM`：确定性离线实现——按角色声明句路由模板，并**回显上下文中的真实数字**，使溯源审计在离线模式下也验证完整链路（CI 与演示均不需要 API key）。

## 6. 公告 RAG（rag/）

- 文档库：巨潮定期报告全文（剔除摘要/英文版，PIT：只暴露 as_of 前已公告的文档），
  PDF 文本按页抽取后落盘缓存；
- 分块：页内滑窗（~700 字、重叠 100），**块不跨页**——保住页码级引用精度；
- 检索：jieba + BM25（零 API 依赖、离线可测、确定性）；embedding 检索为可插拔扩展；
- 问答：引用纪律（每个关键数字标注 文档+页码）+ 复用 fact_check 审计
  （对照语料 = 检索到的片段，所见即所审）。

## 7. 行业对比与基准跑分

- **行业对比分析师**：`--peers` 提供同行代码时启用第五分析师；同行快照与主标的
  同一 PIT 口径（peer_snapshot 复用 load_bundle），对比表进入审计语料；
  单个同行失败降级为四分析师并写 warnings，不阻塞主流程。
- **基准跑分**（`gewu benchmark`）：FinEval 兼容的多选题 harness。仓库不分发基准
  数据（许可证），自带格式样例。多选题准确率衡量底座 LLM 知识，
  与 agent 端到端能力（PIT 回测、溯源率）是两类指标，文档中明确区分。

## 8. 测试策略

- 夹具 = 真实数据快照（2026-06-11 贵州茅台全数据集，544KB parquet），缓存有效期设为∞ → 测试**全离线、确定性**；
- 重点守护不变量：PIT 边界（披露截止日当天 vs 前一天）、指标跟随截断、缓存退化语义、审计契约、辩论轮数、管线端到端。
